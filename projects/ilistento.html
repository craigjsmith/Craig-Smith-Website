i Listen To

Web application to show personalized Spotify listening statistics over time

JavaScript, React, HTML/CSS, Bootstrap, AWS (S3), Spotify API

December 2019

ilistento.net

Source Code

A standard feature amongst classic media players was the ability to see personal listening statistics, such as your top listened to artists and tracks. Spotify lacks this feature; however, this data is available through the Spotify Web API (primarily for use by advertisers). With my web application, users can see their own top 50 artists and tracks of the last 4 weeks, 6 months, and all time.

Built using React, iListenTo.net is accessible on all devices with a web browser; no need for downloads or platform specific code. With Bootstrap, I carefully designed the UI to be responsive to screens of all sizes and to follow Spotify’s branding guidelines. The application authenticates with Spotify using an Implicit-Grant authentication flow, allowing all code to be run client-side and avoiding the need to host a costly backend. I built this application with security in mind, provisioning as few account rights as possible. When users connect their Spotify accounts, the application’s access token is valid only for a single session and all web requests are made client side; none of this information is ever seen by iListenTo.net.

With iListenTo, Spotify users can gain insight into their listening habits and everchanging musical tastes without surrendering this data to third parties.
